内嵌渠道 scene_l2 与主体占比报告

业务建议

产品侧
移动端重点做快编辑：围绕「商品详情页」「社交配图」等高频内容化需求，重点承接快速换背景、调尺寸等轻量编辑，缩短出图路径。
PC / Web 重点做标准化生产和精细化编辑：围绕白底图和标准商品素材处理，重点强化高精度复杂边缘处理、换背景和批量出图能力。
APP头部场景优先突破：设计室 APP 头部场景最集中，优先围绕「商品详情页」「社交配图」「电商主图」优化
推广侧
移动端更适配 C 端场景推广：可从「社交配图」这类C端场景寻找推广机会。
桌面端更适配 B 端场景推广：突出白底图、文档配图等更标准化、更偏生产流程的处理需求。
除商品图外其他机会场景
各端共性需求：社交配图、Logo/图标提取、海报设计在各端都有较大需求。
其他分端场景：APP 端可尝试推广直播带货素材，内嵌/PC / Web 可关注文档配图场景。

各端对比

各端占比结论
  • 移动端（内嵌、APP）「商品详情页」占比 29.7%，高于桌面端（web/PC） 16.5%；「白底/去底」场景移动端占比仅 6.4%，低于桌面端 12.1%，说明移动端更偏轻运营/内容化商品展示，PC/Web 更偏标准商品/去底素材，换背景需求
  • 从场景需求排序看，移动端的「社交配图」场景，尤其是内嵌设计室，需求排名高于桌面端，说明移动端更适合承接社交传播和内容运营类出图场景。
  • 设计室 APP 前三类场景合计 66.6%、偏度 5.13，四端里头部最集中
  • 从 TOP3 二级主体看，移动端更偏人像、服饰和展示型主体，桌面端更偏文档、字体、Logo 和工业件等标准化主体
  • 各端商品图主体存在差异，商品图PC秀秀和web白底图场景 偏工业/零件/工具；而内嵌 / APP 和Web端主图详情图 常见服饰/上装/人像
